# AI Chat Agent Backend

This is a FastAPI backend service that provides a `/chat` endpoint to forward chat messages to a webhook.

## System Integration

This backend acts as a proxy between the Streamlit frontend and the n8n workflow. The data flow is:

1. **Frontend** → Sends chat messages to **Backend** (`/chat` endpoint)
2. **Backend** → Forwards messages to **n8n Workflow** via webhook URL
3. **n8n Workflow** → Processes the message and returns response
4. **Backend** → Forwards response back to **Frontend**

The webhook URL must be configured in the `.env` file and points to your n8n workflow's webhook node.

## Features

- `/chat` endpoint that accepts `sessionId` and `chatInput` parameters
- Forwards requests to the webhook
- Returns the webhook response in a structured format
- Proper error handling for connection failures

## Setup

1. Install dependencies:
```bash
pip install -r requirements.txt
```

2. Create a .env file:
```bash
cp .env.example .env
```

3. Update the .env file with your n8n webhook URL

4. Run the server:
```bash
./run_server.sh
```

## API Endpoints

### POST /chat
Accepts JSON with:
- `sessionId` (string): The session identifier
- `chatInput` (string): The chat message

Returns:
- `message` (string): Status message
- `webhook_response` (array): Response from the webhook

## Testing

You can test the endpoint with curl:
```bash
curl -X POST "http://localhost:8005/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "test_session_123",
    "chatInput": "Hello, this is a test message"
  }'
```

## Error Handling

The endpoint handles the following error cases:
- Connection failures to the webhook
- HTTP errors from the webhook
- Invalid request data
- Internal server errors

## Logging

When the endpoint fails to connect to the webhook, it logs:
- Timestamp of the error
- Error details including the specific error message
- The webhook URL that failed
- Request parameters that were being sent

The error logs are automatically generated by the FastAPI framework and can be viewed in the console where the server is running.
